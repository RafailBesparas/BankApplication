BesparasBank – Software Architecture Overview (Version 3)

1. Purpose

This document describes the high-level architecture of BesparasBank, a secure banking platform built with Spring Boot. It outlines the structure, responsibilities, and modules introduced or improved in Version 3.

2. Technology Stack

  • Backend: Java 17, Spring Boot 3.x, Spring Security
  • Frontend: Thymeleaf, Bootstrap 5, HTML5/CSS3
  • Database: H2 (dev) / MySQL (prod)
  • Messaging (Optional): Apache Kafka

4. Key Components

  • AccountService – Handles core banking logic: deposit, withdrawal, transfer.
  • NotificationService – Sends alerts on low balance, transfers, and system messages.
  • ClientProfileService – Manages user identity and PII information securely.
  • LoanService – (In Progress) Supports loan application and repayment schedule.
  • SecurityConfig – Configures Spring Security with password hashing, login/logout flow.

5. Security Considerations

  • BCrypt used for password encryption.
  • Access to endpoints controlled via Spring Security filters.
  • PII stored in a separate model with limited access exposure.
  • Session-based authentication and CSRF protection enabled.

6. UI Pages

  • /login – User login page
  • /register – New account registration
  • /dashboard – User's financial summary + last actions
  • /transactions – Filterable transaction history
  • /profile – View/edit user info
  • /account-summary – Overview of accounts

7. Documentation Practices

All classes and services are documented with JavaDoc, including:

  • Security annotations and endpoint access details
  • Compliance-aware comments for PII and auditing
  • Clear business rule and purpose documentation
