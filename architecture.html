BesparasBank – Software Architecture Overview

1. Purpose

This document describes the high-level software architecture of BesparasBank, a secure online banking platform built using Spring Boot. It outlines the structure, responsibilities, and integration of core components.

2. Technology Stack

  • Backend: Java 17, Spring Boot, Spring Security
  • Frontend: Thymeleaf, Bootstrap, HTML5/CSS
  • Database: H2 / MySQL (via Spring Data JPA)
  • Security: BCrypt password hashing, authentication providers

4. Key Components

  • AccountService – Handles registration, deposit, withdrawal, and transfer logic.
  • SecurityConfig – Manages authentication, authorization, and password security.
  • ClientProfileService – Manages personal user data (PII) with compliance awareness.
  • Transaction – Represents immutable financial records, used for auditing.

5. Security Considerations

  • All credentials are hashed using BCrypt.
  • All sensitive endpoints are protected using Spring Security annotations and session management.
  • Client PII is separated into ClientProfile with limited exposure.

6. UI Pages

  • /login – User authentication
  • /register – Create account
  • /dashboard – Operations overview
  • /profile – View user information
  • /transactions – View transaction log

7. Documentation Practices

All source files are documented using JavaDoc with special attention to:

  • Business rules & security enforcement
  • Compliance with audit & PII regulations
  • Clear role separation across layers
